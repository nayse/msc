
The primary functional objective of this GUI screen is to allow users to input details for a new financial transaction, either an income or an expense, and then submit or cancel it. 

GUI Intentional Tests:

Test Case 1: Add a new Expense Transaction

Initial State: The "New Transaction" dialog is displayed, with "Enter title" and "Enter amount" fields visible, "Expense" and "Income" radio buttons, and default selections for account ("Main Account") and category ("none"). 
Action 1: Tap on the "Enter title" text field. 
Action 2: Input "Coffee" into the title text field. 
Action 3: Ensure "Expense" radio button is selected (it is by default, but verify). 
Action 4: Tap on the "Enter amount" text field. 
Action 5: Input "5.50" into the amount text field. 
Action 6: (Optional) Tap on the date "2025-06-03" to open the date picker and select a different date, then confirm the date. 
Action 7: (Optional) Tap on the "Account" spinner "Main Account" to change the account if multiple accounts are available. 
Action 8: (Optional) Tap on the "Category" spinner "none" to select a category. 
Action 9: Tap the "SUBMIT" button. 
Expected Result: The "New Transaction" dialog closes, and the new expense transaction "Coffee" with amount "5.50" is successfully added and reflected in the main transaction list.
