# GUI Test Scenarios for Track Recording Application

## Test Scenario 1: Start New Track Recording
**Objective**: Validate the primary function of starting a new GPS track recording

**Test Steps**:
1. Launch the application
2. Verify the main screen loads with empty track list
3. Tap the red FAB button (track_list_fab_action) at coordinates [1160,2658][1384,2882]
4. Verify new track recording interface appears
5. Confirm GPS/location services are activated
6. Validate recording indicators are visible

**Expected Result**: New track recording begins successfully with active GPS tracking

## Edge Case Testing

### Test Scenario 9: No GPS Signal Handling
**Test Steps**:
1. Disable GPS/location services
2. Attempt to start track recording
3. Verify appropriate error handling and user feedback

### Test Scenario 10: Storage Limit Testing
**Test Steps**:
1. Fill device storage to near capacity
2. Attempt to record new tracks
3. Verify graceful handling of storage limitations

### Test Scenario 11: Background App Behavior
**Test Steps**:
1. Start track recording
2. Switch to other apps or lock screen
3. Verify recording continues in background
4. Return to app and check data continuity
