GUI Intentional Tests
Here are GUI intentional tests with logical sequences of user actions to validate the main purposes of the screen.

Test 1: Validate Content Search and Filtering
Objective: Verify that the "Find" bar correctly allows users to enter text, presumably to filter or search the displayed content.

Step	Action	UI Element Tag	Expected Outcome
1	Tap the Find bar.	filterLine	The keyboard should appear, and the cursor should be active in the text field.
2	Enter a search query (e.g., "History").	filterLine	The text "History" should appear in the input field.
3	Observe the countBooks element.	countBooks	The counter (currently "0") should update to reflect the number of items matching the search query (assuming data is present).

Test 2: Validate Content Sorting Functionality
Objective: Verify that the sorting control allows users to change the criteria and order of the displayed content.

Step	Action	UI Element Tag	Expected Outcome
1	Tap the Sort By text (currently "Date").	sortBy	A menu or dialog should appear, listing alternative sorting options (e.g., "Title", "Author", "Size").
2	Select an alternative sort option (e.g., "Title").	N/A (Selection in menu)	The text in the sortBy element should change from "Date" to "Title".
3	Tap the Sort Order Toggle button (up/down arrow).	sortOrder	The icon should change, visually indicating a reversal of the current sort order (e.g., from ascending to descending).

Objective: Verify that the primary navigation tabs successfully switch the user to the corresponding content views.

Step	Action	UI Element Tag	Expected Outcome
1	Tap the Folder tab.	Folder	The content in the main view (recyclerView) should update to display items organized by folder, and the "Folder" tab should become visually highlighted as active.
